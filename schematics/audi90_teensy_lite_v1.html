PHILOSOPHY: Every stage independently testable before connecting to the ECU. Bench validate power → level shifters → firmware → then ECU. The Lite version has fewer stages than Full — you can be at Stage 05 (first ECU connection) in an afternoon.
00

Physical chip read — done ✓

MMS-05C HF201Y43 already read with T48 XGPro. Binary saved at rom_files/physical_reads/893906266D_MMS05C_physical.bin. CRC32: 0x4152E167. This is the recovery file — if anything goes wrong you can re-burn this to a blank 27C256A and the ECU runs as before. Do not proceed without this.

01

Identify ECU 5V regulator

Open ECU, locate the 5V regulator near the power input connector. TO-220 package with metal tab and mounting hole = 7805 or equivalent (≥1A) — proceed with socket power (Path A). Small 3-leg SOT-89 or TO-92 = 78L05 (100mA) — use external LM2596 supply (Path B). Almost certainly a 7805 on the Hitachi 266D board. Write it down for the build notes.

02

Install turned-pin socket in ECU

Desolder MMS-05C from ECU board. Clean pads. Install 28-pin turned-pin DIP socket (gold contacts). Turned-pin contacts give low insertion force and long life vs standard sockets. Verify correct orientation — notch matches original chip orientation. Do not install the Teensy adapter yet.

03

Bench power test — Teensy only

Connect Teensy VIN → 5V bench supply, GND → bench GND. Confirm USB enumeration on laptop. Flash blink sketch from Arduino IDE + Teensyduino. LED on pin 13 should blink 1Hz. This proves the Teensy is alive and programmable. Measure 3.3V on Teensy 3.3V pin = internal LDO working.

04

Level shifter bench test

Wire 74HCT245 ×3 between a DIP28 ZIF socket (as ECU substitute) and Teensy FlexIO pins per the pin map. Flash EPROM emulator firmware with stock ROM image embedded. Use logic analyzer on address/data lines — drive address pins with signal generator, confirm Teensy returns correct data bytes within ~150ns. No ECU connected yet — just the ZIF socket with address/data pins driven manually.

05

SD card test

Format SD card FAT32. Copy tune.bin (physical read binary) and stock.bin (same file, recovery copy) to root. Create maps/ folder, copy map1.bin through map2.bin. Flash firmware — confirm SD mounts on boot (LED steady green = OK, rapid blink = SD error). Confirm map cycling with button on pin 30.

06

First ECU connection — engine off, fuel pump relay out

Remove fuel pump relay as a precaution. Connect Teensy adapter into turned-pin socket in ECU. Teensy powered from socket pin 28 (5V). Ignition on — logic analyzer on address bus should immediately show ECU initiating reads within milliseconds of power-up. Teensy responds with ROM data. Verify timing — ECU expects data within 200ns of address stable. No address mismatches, no bus conflicts.

07

First start — EPROM emulation only

Restore fuel pump relay. Start engine. Should run identically to stock chip — because it IS the stock chip image. If engine runs normally: foundation is confirmed. If rough or no start: EPROM timing issue, return to bench. Once running cleanly on emulated ROM, you can load a modified tune.bin via USB from the tuner app and restart — that's the core Lite workflow.

08

Map switching — validate

With two different tune files on SD as map1.bin and map2.bin (make one with a single byte changed so you can verify it loaded), use the button to switch maps. Confirm LED blink count matches map number. Restart ECU (ignition off/on). Confirm tuner app reports correct ROM CRC for the selected map. You now have a working map switcher.
